# Sentinel 商业化设计文档

> 日期：2026-03-04
> 状态：已批准

## 1. 商业定位

将 Project Sentinel（A股智能投顾哨兵）以两种形态商业化：

1. **OpenClaw Skill 包**：面向技术型投资者，提供源码 + 持仓模板 + 教程
2. **OpenClaw 代安装服务**：面向非技术用户，远程帮装 OpenClaw + 接通推送渠道
3. **套餐**：代安装 + Skill 包 + 持仓定制，一站式交付

### 目标客户（分阶段）

- **Phase 1**：技术圈（开发者、量化爱好者） — Skill 包为主
- **Phase 2**：散户股民 — 简易版 + 代安装为主

### 投入策略

最小化 MVP，不改代码，核心工作是文档、模板、营销内容。快速上架验证需求。

## 5. 合规与风险

### 5.1 法律合规

- **定位**：信息聚合与辅助分析工具，**不是**投资顾问服务
- **免责声明**必须包含：
  - 本工具仅提供信息参考，不构成任何投资建议
  - 投资有风险，用户需自行承担交易决策的全部责任
  - 本工具不持有证券投资咨询牌照
  - AI 分析结果可能存在偏差，不保证准确性
- **不做**：不推荐具体买卖时机、不承诺收益率、不代客操作

### 5.2 安全风险

- 代安装过程中**不保存**客户的 API Key 和 Webhook URL
- 教程中提醒客户不要将 `.env` 文件分享给他人
- 远程操作使用 ToDesk/向日葵，操作完毕后提醒客户修改密码

### 5.3 售后风险

- AkShare API 偶尔不稳定 → 教程中说明，非产品 bug
- Gemini API 限额 → 提醒客户关注 quota
- 飞书 Webhook 过期 → 教程中说明如何更新

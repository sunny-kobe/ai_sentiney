# OpenClaw 安全部署服务商业化方案

> 日期：2026-03-04
> 状态：修订中（v3，基于市场调研 + 置信度分层修正）

## 0. 定位说明

**核心业务：OpenClaw 安全部署与配置服务**
- 帮非技术用户远程安装 OpenClaw、接通国内消息渠道（飞书/钉钉）、配置 AI 模型
- **核心差异化：安全加固 + 省钱配置**，而非单纯"帮你装上"
- 本质上是"装系统 + 装杀毒 + 调优"的组合服务

**附属产品：Skill 安装与配置**
- 客户有需要时，可额外安装日程管理、翻译、知识库等通用 Skill
- Skill 安装是纯技术服务，定位清晰

> **关于 Sentinel（A 股分析 Skill）**
> - **不在任何公开销售环节出现**，零营销
> - 监管案例持续将无资质荐股/投顾认定为违规（见[证监会厦门局案例](http://www.csrc.gov.cn/xiamen/c7602409/content.shtml)、[基金业协会文件](https://www.amac.org.cn/governmentrules/zcgz_jcxgf/jcxgf_fgxwj/202003/t20200311_7418.html)）
> - 2025 年投顾机构收到 74 张罚单（[证券时报统计](https://www.stcn.com/article/detail/3539073.html)），AI 包装荐股的执法力度在加强
> - Sentinel 输出的 SAFE/DANGER/WATCH 信号实质接近证券投资咨询，免责声明不改变行为性质
> - 如有客户主动问起，仅定位为"开源数据看板安装"，不提"分析""推荐""信号"

## 2. 差异化策略：安全加固为核心

纯安装 ¥30 就有人做。要卖到 ¥199-399，必须给客户感知到的额外价值。

**安全加固是最强差异化**，理由：
- 数千个 OpenClaw 实例暴露在公网，无认证、默认端口
- 已出现恶意 Skill 窃取 API Key 的事件
- OpenClaw 默认心跳机制（[官方文档](https://docs.openclaw.ai/reference/token-use)）会持续消耗 token，实际成本取决于模型选择、心跳频率和上下文大小，配置不当时可能显著增费
- 这些问题是**云平台一键部署做不到**的（装完就不管了）

| 差异化点 | 具体做法 | 竞品缺失 |
|----------|----------|----------|
| **安全加固**（核心） | 反向代理 + 认证配置 + 防暴露检查 | 大量 ¥30 安装无安全意识 |
| **省钱配置**（核心） | 心跳频率优化 + 模型路由 + token 用量控制 | 无人关注，但客户痛点明确 |
| **国内渠道专精** | 飞书/钉钉接入（非 Telegram/Discord） | 多数教程只教海外渠道 |
| **Skill 配置** | 帮装实用 Skill 并调通 | 纯安装不管 Skill |
| **安全审查** | 检查已安装 Skill 是否含恶意代码 | 无人做 |
| **中文全程服务** | 中文教程 + 中文售后 | 官方文档英文为主 |

**核心话术：装完就裸奔？我帮你装好、装安全、省钱跑。**

## 6. 合规与风险

### 6.1 法律合规

代安装服务是纯技术服务，**不涉及证券投资咨询牌照**。

关于金融类 Skill（Sentinel 等）：
- **不在公开渠道销售**，不做任何营销推广
- 监管案例持续将无资质荐股/投顾认定为违规：
  - [证监会厦门局](http://www.csrc.gov.cn/xiamen/c7602409/content.shtml)：非法证券投资咨询认定标准
  - [基金业协会](https://www.amac.org.cn/governmentrules/zcgz_jcxgf/jcxgf_fgxwj/202003/t20200311_7418.html)：投顾行为规范
  - [证券时报](https://www.stcn.com/article/detail/3539073.html)：2025 年投顾机构收到 74 张罚单
- AI 包装荐股的执法力度在加强，免责声明不改变行为实质
- 闲鱼上"AI + 股票"关键词可能触发自动审核删帖

### 6.2 安全风险

- 代安装过程中**不保存**客户的 API Key 和 Webhook URL
- 远程操作使用 ToDesk/向日葵，操作完毕后提醒客户修改密码
- 安装时主动帮客户做安全加固（这本身是核心卖点）

### 6.3 售后风险

- OpenClaw 更新导致配置失效 → 提供 7 天免费售后（订阅用户持续覆盖）
- API 限额问题 → 教程中说明，非服务范围
- 渠道 token 过期 → 教程中说明如何更新

### 6.4 市场风险

| 风险 | 概率 | 应对 |
|------|------|------|
| 价格战（降到 ¥30 以下） | 高 | 靠安全加固 + 省钱配置差异化，不打价格战 |
| 云平台一键部署普及 | 高 | 转向"已有实例的安全体检 + 运维订阅"（一键部署恰恰不管这些） |
| OpenClaw 热度消退 | 中 | 技能可迁移到下一个 AI 工具浪潮 |
| 恶意 Skill 事件爆发 | 中 | 反而利好——安全审查服务需求上升 |

## 附录：数据置信度说明

本方案中引用的数据按置信度分三级：

- **高**：可直接验证的一手数据（GitHub stars、官方统计、政府文件、平台官方文档）
- **中**：主流媒体调查报道、开发者社区内容（可参考但非硬证据）
- **低**：行业报告摘要、未核实原始出处的二手引用（仅作背景参考）

决策应主要基于高置信度数据，中置信度数据作为辅助判断，低置信度数据不作为关键论据。

# AI Agent 协同工作流指南 v3（官方推荐版）

> 基于 Claude Code 和 Hermes Agent 官方推荐的最佳实践

## 核心原则

**简单、清晰、验证**

---

## 工作流（3 步）

```
1. 配置 → CLAUDE.md + AGENTS.md
2. 执行 → 给清晰指令，让 Claude 自主执行
3. 验证 → 运行测试，检查结果
```

---

## 第 1 步：配置

### CLAUDE.md（必须）

在项目根目录创建 `CLAUDE.md`，包含：

```markdown
# 项目概述
[一句话描述]

# 常用命令
- 测试: python -m pytest tests/ -q
- 启动: python -m src.main

# 代码规范
- [关键规则]

# 关键文件
- [重要文件列表]
```

### AGENTS.md（可选）

给 Codex 用，内容类似 CLAUDE.md。

---

## 第 2 步：执行

### 给清晰指令

**好的指令**：
```
"修复 src/xxx.py 的 bug，运行测试验证"
"添加 XXX 功能，覆盖正常/边界/错误场景"
"重构 XXX，保持 API 不变，所有测试通过"
```

**坏的指令**：
```
"优化一下代码"（太模糊）
"改改看"（没有目标）
"随便改改"（没有验收标准）
```

### 让 Claude 自主执行

Claude Code 会自动：
1. 读 CLAUDE.md 了解项目
2. 探索代码库
3. 执行任务
4. 运行测试

**不需要**：
- 手动指定用哪个模型
- 手动判断 TDD 还是直接
- 手动管理并行任务

---

## 第 3 步：验证

### 运行测试

```bash
python -m pytest tests/ -q
```

### 检查结果

- 测试通过？
- 功能正确？
- 代码质量？

### 提交

```bash
git add -A
git commit -m "feat: 描述"
```

---

## 什么时候用 Claude Code？

| 场景 | 用 Claude Code | 自己做 |
|------|---------------|--------|
| 多文件修改 | ✅ | ❌ |
| 需要写测试 | ✅ | ❌ |
| 简单配置修改 | ❌ | ✅ |
| 1-2 行代码 | ❌ | ✅ |

**判断标准**：超过 2 个文件 → 用 Claude Code

---

## 任务模板（简化版）

```markdown
## 目标
[一句话]

## 相关文件
- src/xxx.py
- tests/test_xxx.py

## 验收标准
- [ ] 所有测试通过
```

**不需要**：
- 约束（Claude Code 会读 CLAUDE.md）
- 详细的分步指令（Claude Code 会自己规划）

---

## 常见问题

### Q: 什么时候用 TDD？
**A**: 官方没强调。需要时再用，不用默认走 TDD。

### Q: 什么时候并行？
**A**: 官方说用子 agent。有 3+ 独立任务时再考虑。

### Q: 怎么控制成本？
**A**: 官方没强调。用户说省 token 时再考虑。

### Q: 错误了怎么办？
**A**: 官方说"迭代修复"。让 Claude 自己修复，失败了再介入。

---

## 总结

**核心 3 步**：
1. 配置（CLAUDE.md）
2. 执行（清晰指令）
3. 验证（测试通过）

**不需要**：
- 复杂的模式判断
- 详细的成本控制
- 预设的错误恢复策略

**记住**：简单、清晰、验证
